Next.js အခြေခံမှ အသေးစိတ်အထိ

Web Development ကို ပိုမိုမြန်ဆန်၊ ကောင်းမွန်အောင် တည်ဆောက်ခြင်း


တင်ဆက်သူ: Front-end Expert

Next.js ဆိုတာ ဘာလဲ?

Next.js ဆိုတာ React Framework တစ်ခုဖြစ်ပါတယ်။ Vercel ကုမ္ပဏီကနေ ဖန်တီးထားတာဖြစ်ပြီး Web Application တွေကို Production အဆင့်အထိ အလွယ်တကူထုတ်လုပ်နိုင်အောင် ကူညီပေးပါတယ်။

  • React ရဲ့ အားနည်းချက်တွေကို ဖြည့်ဆည်းပေးတယ်။
  • Full-stack web application တွေရေးဖို့ အဆင်ပြေတယ်။
  • Speed နဲ့ Performance အတွက် အထူးတည်ဆောက်ထားတယ်။

React နဲ့ Next.js ဘာကွာလဲ?

React (Library)

UI (User Interface) တည်ဆောက်ဖို့အတွက်ပဲ အဓိကသုံးပါတယ်။ Routing, Data fetching တွေအတွက် တခြား Library တွေ (ဥပမာ- React Router) ထပ်လိုပါတယ်။

Next.js (Framework)

React ကိုအခြေခံထားပေမယ့် Routing, API, SEO, Server-side rendering အားလုံး အသင့်ပါဝင်ပြီးသား All-in-one package ဖြစ်ပါတယ်။

ဘာကြောင့် Next.js ကို သုံးသင့်တာလဲ?

  • အမြန်နှုန်း (Performance): Web page တွေ လျှပ်တစ်ပြက်ပွင့်ဖို့ ကူညီပေးတယ်။
  • SEO ကောင်းမွန်ခြင်း: Search Engine တွေ (Google) မှာ ရှာရလွယ်အောင် အထောက်အပံ့ပေးတယ်။
  • Developer Experience: Setup လုပ်ရလွယ်ပြီး Code ရေးရတာ မြန်ဆန်စေတယ်။

Next.js ရဲ့ အဓိက လုပ်ဆောင်ချက်များ (Core Features)

  • Server-Side Rendering (SSR)
  • Static Site Generation (SSG)
  • App Router & File-based Routing
  • Image & Font Optimization
  • API Routes
  • Built-in Security

Server-Side Rendering (SSR) ဆိုတာဘာလဲ?

User က Website ကို ဝင်ကြည့်တဲ့ အချိန်တိုင်းမှာ Server ကနေ HTML page ကို ချက်ချင်းတည်ဆောက်ပြီး Browser ဆီပို့ပေးတဲ့ စနစ်ပါ။

  • အားသာချက်: အမြဲတမ်း Data အသစ် (Up-to-date) ရတယ်။ SEO အတွက် အလွန်ကောင်းတယ်။
  • သုံးသင့်တဲ့နေရာ: E-commerce Cart, Dashboard, Social Media Feed စတဲ့ Data အမြဲပြောင်းလဲနေတဲ့ နေရာတွေ။

Static Site Generation (SSG)

Website ကို Server ပေါ်တင်လိုက်ကတည်းက (Build time) HTML page တွေကို ကြိုတင်တည်ဆောက်ထားတဲ့ စနစ်ပါ။

  • အားသာချက်: အလွန်မြန်ဆန်တယ်။ Server အလုပ်လုပ်ရသက်သာတယ်။
  • သုံးသင့်တဲ့နေရာ: Blog post တွေ, Documentation တွေ, Portfolio website တွေ (Data ခဏခဏ မပြောင်းတဲ့နေရာတွေ)။

Client-Side Rendering (CSR)

Browser (Client) ကနေပဲ JavaScript တွေကို Run ပြီးမှ Page ကိုပေါ်စေတဲ့ စနစ်ပါ။ (React ရဲ့ မူလအလုပ်လုပ်ပုံ)

  • အားသာချက်: Page ပွင့်ပြီးသွားရင် နောက် Page တွေကိုကူးတဲ့အခါ အရမ်းမြန်တယ်။
  • အားနည်းချက်: ပထမဆုံးအကြိမ် Load လုပ်ချိန်ကြာနိုင်တယ်။ SEO အတွက် မကောင်းဘူး။

Incremental Static Regeneration (ISR)

SSG နဲ့ SSR ရဲ့ ကောင်းကွက်တွေကို ပေါင်းထားတာပါ။ Page ကို ကြိုတင် Build ထားပေမယ့်၊ သတ်မှတ်ထားတဲ့ အချိန်ရောက်တိုင်း နောက်ကွယ်ကနေ (Background) Data အသစ်ပြန်ယူပြီး Page ကို Update လုပ်ပေးပါတယ်။

ဥပမာ - သတင်း Website တွေမှာ ၆၀ စက္ကန့်တစ်ခါ သတင်းအသစ် ပြန်ဆွဲယူတာမျိုး။

ဘယ်အချိန်မှာ ဘယ် Rendering ကိုသုံးမလဲ?

Rendering အချိန် (When to generate) သင့်တော်သော နေရာ (Use Case)
SSG Build Time Blogs, Marketing pages, Help docs
SSR Request Time (အချိန်တိုင်း) Dynamic Dashboards, Shopping Cart
ISR Background (သတ်မှတ်ချိန်တိုင်း) News feeds, Product Listings
CSR On the Browser Private routes, highly interactive UI

File-based Routing စနစ်

Next.js ရဲ့ အလွယ်ကူဆုံး အားသာချက်ပါ။ Folder နဲ့ File နာမည်ပေးလိုက်တာနဲ့ အလိုအလျောက် URL Route တွေ ဖြစ်သွားပါတယ်။

app/
├── page.tsx     --> yoursite.com/
├── about/
│ └── page.tsx  --> yoursite.com/about
└── blog/
└── page.tsx  --> yoursite.com/blog

App Router သစ် (Next.js 13+)

ယခင်က pages/ folder ကိုသုံးပေမယ့် အခု app/ folder စနစ် (App Router) ကို ပြောင်းလဲအသုံးပြုလာပါပြီ။

  • React Server Components ကို Default အဖြစ်သုံးလာတယ်။
  • Layouts (layout.tsx) တွေကို အလွယ်တကူ သတ်မှတ်နိုင်တယ်။
  • Loading, Error တွေကို ဖိုင်ခွဲပြီး ရေးရတာ ပိုရှင်းလင်းသွားတယ်။

Next.js နှင့် SEO (Search Engine Optimization)

Google က Website တွေကို ဖတ်တဲ့အခါ HTML အပြည့်အစုံကို လိုချင်ပါတယ်။ Next.js ရဲ့ SSR နဲ့ SSG က HTML ကို အသင့်ထုတ်ပေးလို့ SEO အတွက် အထူးကောင်းမွန်ပါတယ်။

// app/layout.tsx တွင် Meta data ထည့်ခြင်း export const metadata = { title: 'My Next.js Blog', description: 'Learn Next.js easily in Burmese', }

ပုံများအတွက် Image Optimization

<img> Tag အစား Next.js ရဲ့ <Image> component ကိုသုံးခြင်းဖြင့် ပုံတွေကို အလိုအလျောက် Size ချုံ့ပေးခြင်း၊ WebP format ပြောင်းပေးခြင်းများ လုပ်ဆောင်ပေးပါတယ်။

import Image from 'next/image' export default function Page() { return ( <Image src="/profile.png" width={500} height={500} alt="Profile Picture" /> ) }

Font များကို စီမံခန့်ခွဲခြင်း (next/font)

Google Fonts တွေကို လွယ်ကူစွာ ချိတ်ဆက်နိုင်ပြီး၊ နှောင့်နှေးမှုမရှိအောင် (No Layout Shift) အလိုအလျောက် Optimize လုပ်ပေးပါတယ်။

import { Inter } from 'next/font/google' const inter = Inter({ subsets: ['latin'] }) export default function RootLayout({ children }) { return ( <html lang="en" className={inter.className}> <body>{children}</body> </html> ) }

Backend API များကို Next.js တွင်ရေးခြင်း

Next.js မှာ Backend Server သက်သက်ထောင်စရာမလိုဘဲ API (Route Handlers) တွေကိုပါ တစ်ခါတည်း ရေးသားနိုင်ပါတယ်။

// app/api/hello/route.ts export async function GET() { return Response.json({ message: 'Hello from Next.js API!' }) }

ဒါကြောင့် Next.js ကို Full-stack Framework လို့ ခေါ်ကြတာပါ။

မစတင်ခင် ဘာတွေသိထားဖို့လိုမလဲ?

  • HTML, CSS အခြေခံ
  • JavaScript (ES6+)
  • React အခြေခံ
    • Components
    • Props & State
    • Hooks (useState, useEffect)

React ကို မသိဘဲ Next.js ကို တိုက်ရိုက်လေ့လာဖို့ မခက်ခဲပေမယ့်၊ React အခြေခံရှိထားရင် ပိုမြန်မြန် တတ်မြောက်နိုင်ပါတယ်။

၁။ Node.js ထည့်သွင်းခြင်း

Next.js အသုံးပြုဖို့ ကွန်ပျူတာမှာ Node.js install လုပ်ထားဖို့လိုပါတယ်။

  • Website: nodejs.org သို့သွားပါ။
  • LTS (Long Term Support) Version ကို ဒေါင်းလုဒ်ဆွဲပြီး Install လုပ်ပါ။
  • စစ်ဆေးရန် Terminal (CMD) တွင် ရိုက်ထည့်ပါ:
    node -v
    npm -v

၂။ Project အသစ်စတင်ခြင်း

Terminal ဖွင့်ပြီး အောက်ပါ Command ကို ရိုက်ထည့်ပါ။

npx create-next-app@latest my-app

မေးခွန်းလေးတွေ မေးပါလိမ့်မယ်...

  • Use TypeScript? -> Yes/No
  • Use Tailwind CSS? -> Yes (Recommended)
  • Use App Router? -> Yes (Must use)

၃။ Project ဖိုင်တည်ဆောက်ပုံ

  • 📁 app/ - Main code တွေရေးရမယ့် နေရာ (Routing)
  • 📄 app/layout.tsx - Website တစ်ခုလုံးရဲ့ အပြင်အဆင်
  • 📄 app/page.tsx - Home Page (ပထမဆုံးမြင်ရမယ့်စာမျက်နှာ)
  • 📁 public/ - ပုံတွေ၊ Font တွေ (Static Assets) ထားတဲ့နေရာ
  • 📄 package.json - Project ရဲ့ Library မှတ်တမ်း

၄။ Development Server Run ခြင်း

Project folder ထဲဝင်ပြီး Server စတင် Run ပါမည်။

cd my-app npm run dev

Browser ဖွင့်ပြီး http://localhost:3000 ကိုသွားကြည့်ပါ။ Next.js ရဲ့ Welcome page ကို မြင်ရပါမည်။

၅။ ပထမဆုံး Page အသစ်ဖန်တီးခြင်း

ဥပမာ - About page ဖန်တီးလိုပါက `app/about/page.tsx` ဖိုင်ကို တည်ဆောက်ပါ။

// app/about/page.tsx export default function AboutPage() { return ( <main> <h1>ကျွန်ုပ်တို့အကြောင်း</h1> <p>Next.js လေ့လာနေပါသည်။</p> </main> ) }

http://localhost:3000/about တွင် သွားကြည့်နိုင်ပါပြီ။

၆။ Page များချိတ်ဆက်ခြင်း (Link)

HTML ရဲ့ <a> tag အစား Next.js ရဲ့ <Link> ကိုသုံးရပါမယ်။ (Page မပွင့်ဘဲ ချက်ချင်း ကူးသွားစေရန်)

import Link from 'next/link' export default function Home() { return ( <div> <Link href="/about"> About Page သို့သွားရန် </Link> </div> ) }

၇။ Styling (ဒီဇိုင်းပြင်ဆင်ခြင်း)

Next.js မှာ CSS ရေးဖို့ နည်းလမ်းများစွာ ရှိပါတယ်။

  • Global CSS: app/globals.css တွင်ရေးခြင်း။
  • CSS Modules: styles.module.css ဖိုင်ခွဲရေးခြင်း (Class name တွေ မထပ်အောင် ကာကွယ်ပေးသည်)။
  • Tailwind CSS: အကောင်းဆုံးနှင့် လက်ရှိ လူသုံးအများဆုံး (Setup လုပ်ရာတွင် အသင့်ရွေးချယ်ခဲ့ပါက တန်းသုံးနိုင်သည်)။

၈။ Server vs Client Components

Server Component (Default)

Server မှာပဲ အလုပ်လုပ်တယ်။ Database ကို တိုက်ရိုက်ချိတ်လို့ရတယ်။ onClick လိုမျိုး Event တွေ သုံးလို့မရဘူး။

Client Component

Browser မှာ အလုပ်လုပ်တယ်။ useState, useEffect နဲ့ onClick တွေသုံးချင်ရင် ဖိုင်ထိပ်ဆုံးမှာ 'use client' လို့ ကြေညာပေးရမယ်။

၉။ Data Fetching (Data ရယူခြင်း)

Server Components တွေမှာ async/await ကို တိုက်ရိုက်သုံးပြီး API ကနေ Data လှမ်းယူလို့ရပါတယ်။

export default async function Page() { // Server ဘက်မှာပဲ run ပါမယ် const res = await fetch('https://api.example.com/data'); const data = await res.json(); return ( <ul> {data.map(item => <li key={item.id}>{item.name}</li>)} </ul> ); }

၁၀။ Loading & Error ပြသခြင်း

Data ဆွဲယူနေချိန်နဲ့ အမှားအယွင်းဖြစ်ချိန်တွေအတွက် UI ကို လွယ်လွယ်ကူကူ ဖန်တီးနိုင်ပါတယ်။

  • loading.tsx - Data ယူနေချိန်မှာ အလိုအလျောက် ပြသပေးမည့် ဖိုင်။
  • error.tsx - Error တက်သွားရင် Website တစ်ခုလုံး ပျက်မသွားဘဲ ဒီဖိုင်ကိုပဲ ပြသပေးပါမည်။

၁၁။ Deployment (Website လွှင့်တင်ခြင်း)

Next.js ကို ဖန်တီးထားတဲ့ Vercel မှာ အလွယ်ကူဆုံး Free လွှင့်တင်နိုင်ပါတယ်။

  1. မိမိ Code ကို GitHub (သို့) GitLab တွင် တင်ပါ။
  2. Vercel.com တွင် အကောင့်ဖွင့်ပါ။
  3. GitHub repository ကို Vercel နှင့် ချိတ်ဆက်ပါ။
  4. "Deploy" ကိုနှိပ်လိုက်ရုံဖြင့် အလိုအလျောက် Build လုပ်ပြီး Live URL ရရှိပါမည်။

အနှစ်ချုပ် (Conclusion)

  • Next.js သည် React ၏ အဆင့်မြင့် Framework တစ်ခုဖြစ်သည်။
  • Performance, SEO နှင့် Developer Experience ကို အကောင်းဆုံး ထောက်ပံ့ပေးသည်။
  • SSR, SSG, API route စသည်တို့ပါဝင်၍ Full-stack အဖြစ်သုံးနိုင်သည်။
  • ခေတ်သစ် Web Development လောကတွင် မရှိမဖြစ် လေ့လာထားသင့်သော နည်းပညာတစ်ခု ဖြစ်ပါသည်။
